## 1. 需求概述

### 1.1 项目背景

用户端是蛋糕系统面向消费者的核心模块，为用户提供完整的在线购物体验，包括商品浏览、购物车管理、订单下单、支付及个人信息维护等功能，是连接用户与商家的重要桥梁。

### 1.2 目标用户

- **普通消费者**: 浏览商品、购买蛋糕的终端用户

### 1.3 核心价值

- 便捷的商品浏览和搜索体验
- 流畅的购物车和下单流程
- 清晰的订单跟踪和管理
- 完善的个人信息和收货地址管理

## 3. 用户注册登录模块

### 3.1 用户注册

**功能描述**: 新用户通过填写注册信息创建账号

**注册字段**:

| 字段名 | 类型 | 必填 | 说明 |
| :--- | :--- | :-: | :--- |
| 用户名 | 文本 | 是 | 4-20个字符，支持字母、数字、下划线 |
| 手机号 | 文本 | 是 | 11位有效手机号码 |
| 密码 | 文本 | 是 | 6-20位字符，建议包含字母和数字 |
| 确认密码 | 文本 | 是 | 与密码保持一致 |

**校验规则**:
- 用户名不可与已有用户重复
- 手机号不可与已有用户重复
- 密码长度不少于6位
- 两次输入的密码必须一致

### 3.2 用户登录

**功能描述**: 已注册用户通过账号密码登录系统

**登录方式**:
- 用户名 + 密码
- 手机号 + 密码

**功能特性**:
- 登录失败提示具体原因（账号不存在、密码错误）
- 登录成功后保存用户登录状态（Session）
- 记住登录状态（可选）

### 3.3 退出登录

**功能描述**: 用户主动退出当前登录状态

**操作流程**:
1. 用户点击"退出登录"按钮
2. 系统弹出确认提示
3. 用户确认后清空登录状态
4. 跳转至登录页或首页

### 3.4 密码修改

**功能描述**: 已登录用户修改自己的登录密码

**操作流程**:
1. 输入原密码进行身份验证
2. 输入新密码
3. 再次输入新密码确认
4. 验证通过后更新密码

**校验规则**:
- 原密码必须正确
- 新密码长度不少于6位
- 新密码不能与原密码相同

## 4. 商品浏览模块

### 4.1 首页展示

**功能描述**: 用户进入系统首页，展示核心内容区域

**页面结构**:

| 区域 | 内容说明 |
| :--- | :--- |
| 顶部搜索栏 | 搜索框、购物车入口 |
| 轮播图 | 活动宣传、新品推荐等轮播展示 |
| 分类导航 | 快捷分类入口图标 |

- 商品图片

- 商品价格

- 默认排序

- 按新品优先

### 4.3 商品详情

**功能描述**: 展示单个商品的完整信息

**详情内容**:

| 信息项 | 说明 |
| :--- | :--- |

| 商品名称 | 商品全称 |
| 商品价格 | 销售价格 |

- 加入购物车

**功能描述**: 展示最新上架的商品

**功能特性**:

- 支持多级分类（如有）

## 5. 购物车模块

### 5.1 加入购物车

**功能描述**: 将商品添加到购物车中

**操作入口**:
- 商品详情页：选择数量后点击"加入购物车"
- 商品列表：快速加入按钮

**业务规则**:
- 同一商品多次加入时，数量累加
- 加入数量不能超过库存数量
- 加入成功后给出提示（toast）
- 购物车角标实时更新数量

### 5.2 数量增减

**功能描述**: 调整购物车中商品的购买数量

**操作方式**:
- "+"按钮：数量+1
- "-"按钮：数量-1
- 直接输入数量

**校验规则**:
- 数量最小值为1
- 数量最大值为商品库存
- 数量为1时，点击"-"弹出删除确认

### 5.3 删除商品

**功能描述**: 将商品从购物车中移除

**操作方式**:
- 左滑删除
- 勾选批量删除
- 数量减到0时确认删除

**操作流程**:
1. 用户点击删除操作
2. 弹出确认对话框
3. 用户确认后从购物车移除
4. 更新购物车列表和总价

### 5.4 总价计算

**功能描述**: 实时计算选中商品的总金额

**计算规则**:
- 总价 = Σ(商品单价 × 商品数量)
- 只计算勾选状态的商品
- 价格保留两位小数
- 数量变化时实时更新

### 5.5 库存校验

**功能描述**: 校验购物车商品的库存状态

**校验时机**:
- 加入购物车时
- 调整数量时
- 进入购物车页面时
- 结算提交订单时

**异常处理**:
- 库存不足时禁用勾选和结算按钮
- 显示"库存不足"提示
- 超出库存时自动调整为最大库存数量

### 5.6 选中结算

**功能描述**: 勾选需要购买的商品进行结算

**功能特性**:
- 单选 / 全选
- 选中商品数量统计
- 选中商品总金额统计
- 结算按钮跳转到确认订单页

## 6. 订单管理模块

### 6.1 创建订单

**功能描述**: 用户提交订单信息，生成订单

**订单信息**:

| 信息项 | 说明 |
| :--- | :--- |
| 收货地址 | 从地址列表选择，默认选中默认地址 |
| 商品明细 | 从购物车带入的商品列表 |
| 商品总金额 | 所有商品金额合计 |
| 配送费 | 配送费用（0为免配送费） |
| 订单总额 | 商品总金额 + 配送费 |
| 订单备注 | 用户填写的备注信息（选填） |

**操作流程**:
1. 从购物车或商品详情进入确认订单页
2. 选择或新增收货地址
3. 确认商品信息和金额
4. 填写订单备注（可选）
5. 点击"提交订单"
6. 生成订单，状态为"待付款"
7. 跳转至支付页面

### 6.2 订单支付

**功能描述**: 对"待付款"状态的订单进行支付

**支付方式**:
- 微信支付
- 支付宝支付
- 余额支付（可选）

**操作流程**:
1. 进入支付页面，显示订单金额
2. 选择支付方式
3. 点击"确认支付"
4. 调用第三方支付接口
5. 支付成功后订单状态变为"已付款(待发货)"
6. 跳转至订单详情页

### 6.3 订单列表

**功能描述**: 查看当前用户的所有订单

**状态Tab**:
- 全部
- 待付款
- 待发货（已付款）
- 配送中
- 已完成
- 已取消

**订单卡片信息**:

| 信息项 | 说明 |
| :--- | :--- |
| 订单号 | 唯一订单标识 |
| 下单时间 | 订单创建时间 |
| 订单状态 | 当前订单状态 |
| 商品缩略图 | 订单内商品图片（最多显示3-4张） |
| 商品数量 | 订单商品总件数 |
| 订单金额 | 订单实付金额 |
| 操作按钮 | 根据状态显示不同操作 |

### 6.4 订单详情

**功能描述**: 查看订单的完整详细信息

**详情内容**:

| 区域 | 信息项 |
| :--- | :--- |
| 订单状态 | 当前状态、状态进度条 |
| 收货信息 | 收货人、手机号、详细地址 |
| 商品明细 | 商品图片、名称、单价、数量、小计 |
| 费用明细 | 商品总额、配送费、实付金额 |
| 订单信息 | 订单号、下单时间、支付时间、发货时间、完成时间 |
| 订单备注 | 用户备注 |

### 6.5 订单搜索

**功能描述**: 根据订单号搜索订单

**搜索规则**:
- 支持订单号精确或模糊搜索
- 搜索结果按时间倒序排列
- 无结果时提示"未找到相关订单"

### 6.6 取消订单

**功能描述**: 取消"待付款"状态的订单

**操作流程**:
1. 在订单列表或详情页点击"取消订单"
2. 弹出取消原因选择
3. 用户选择取消原因并确认
4. 订单状态变为"已取消"
5. 库存回滚（如有锁定）

**限制条件**:
- 只有"待付款"状态的订单可以取消
- 已付款订单需联系客服取消

### 6.7 确认收货

**功能描述**: 用户确认已收到商品

**操作流程**:
1. 在订单列表或详情页点击"确认收货"
2. 弹出确认提示
3. 用户确认后订单状态变为"已完成"
4. 跳转至订单详情页

| 字段名 | 类型 | 必填 | 说明 |

- 最多可保存10个收货地址
- 默认地址有且仅有一个

**功能描述**: 已登录用户修改登录密码

**操作流程**:
1. 输入原密码
2. 输入新密码
3. 确认新密码
4. 点击确认修改
5. 验证通过后更新密码，跳转登录页重新登录

**校验规则**:
- 原密码必须正确
- 新密码长度6-20位
- 新密码不能与原密码相同
- 两次输入的新密码必须一致

## 8. 业务流程

### 8.1 注册登录流程

```
新用户 → 点击注册 → 填写信息 → 验证通过 → 注册成功 → 自动登录 → 首页
                                              ↓
                                          注册失败 → 提示错误信息

已注册用户 → 点击登录 → 输入账号密码 → 验证通过 → 登录成功 → 首页
                                      ↓
                                  登录失败 → 提示错误原因
```

### 8.2 购物下单流程

```
浏览商品 → 加入购物车 → 进入购物车 → 勾选商品 → 去结算
   ↓                                            ↓
立即购买 →───┐                           确认订单信息
             ↓                                 ↓
         确认订单页 → 选择收货地址 → 提交订单 → 生成订单(待付款)
                                                    ↓
                                              立即支付 ──→ 支付成功 → 订单状态:已付款(待发货)
                                                    ↓
                                              稍后支付 → 订单列表中找到订单再支付
```

### 8.3 订单状态流转

```
创建订单 → 待付款 → 支付成功 → 已付款(待发货) → 商家发货 → 配送中 → 确认收货 → 已完成
              ↓                      ↓
            取消订单               申请取消
              ↓                      ↓
            已取消                 已取消
```

| 普通用户 | 查看和管理自己的购物车 |
| 普通用户 | 查看自己的订单、下单、支付、取消订单 |
| 普通用户 | 浏览所有上架商品 |

**数据隔离原则**:
- 用户只能访问自己的数据
- 购物车数据按用户隔离
- 订单数据按用户隔离
- 收货地址数据按用户隔离
- 个人信息仅本人可查看和修改

## 11. 非功能需求

| 需求类型 | 描述 |

| 安全性 | 用户密码加密存储，敏感操作需验证身份，防止越权访问 |

## 12. 附录：订单状态说明

### 12.1 订单状态列表

| 状态码 | 状态名称 | 说明 | 用户可操作 |
| :-: | :--- | :--- | :--- |
| 1 | 待付款 | 订单已创建，等待用户支付 | 去支付、取消订单 |
| 2 | 已付款(待发货) | 用户已支付，等待商家发货 | 查看详情、申请取消 |
| 3 | 配送中 | 商品已发货，配送中 | 查看详情、确认收货 |
| 4 | 已完成 | 用户已确认收货，订单完成 | 查看详情、再次购买 |
| 5 | 已取消 | 订单已取消 | 查看详情、删除订单 |
