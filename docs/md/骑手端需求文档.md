## 1. 项目概述

### 1.1 项目背景

骑手端是蛋糕系统的重要组成部分，为配送骑手提供订单接收、配送管理、路线导航等核心功能，实现订单配送全流程管理。

### 1.2 目标用户

- **配送骑手**: 负责订单的配送工作

### 1.3 核心价值

- 高效接收和处理配送订单
- 实时更新配送状态
- 提供导航路线指引
- 统计配送业绩数据

## 3. 订单配送模块

### 3.1 订单列表

**功能描述**: 展示骑手相关的所有订单，按状态分类

**页面结构**:

- 底部Tab切换：待接单、待配送、已完成
- 订单卡片：订单号、商品数量、配送地址、预计送达时间、操作按钮

**订单卡片信息**:

| 信息项    | 说明         |
| :----- | :--------- |
| 订单号    | 唯一订单标识     |
| 商品数量   | 订单商品总数     |
| 配送地址   | 收货地址简要信息   |
| 预计送达时间 | 系统计算的预计时间  |
| 订单金额   | 订单总额（用于参考） |

### 3.2 接单功能

**功能描述**: 骑手接收待配送订单

**操作流程**:

1. 在待接单列表中选择订单
2. 点击"接单"按钮
3. 系统确认接单成功
4. 订单状态变为待取货

### 3.3 订单详情

**功能描述**: 展示订单完整信息

**信息结构**:

- 订单基本信息：订单号、下单时间、订单状态
- 收货信息：收货人、电话、详细地址
- 商品明细：商品名称、数量
- 配送信息：配送费、预计送达时间

### 3.4 取货确认

**功能描述**: 骑手到店后确认取货

**操作流程**:

1. 到达商家门店
2. 点击"确认取货"按钮
3. 订单状态更新为待配送
4. 开始导航到收货地址

### 3.5 开始配送

**功能描述**: 更新订单状态为配送中

**操作流程**:

1. 取货完成后自动进入配送状态
2. 或手动点击"开始配送"
3. 系统记录配送开始时间

### 3.6 送达确认

**功能描述**: 确认订单已送达

**操作流程**:

1. 到达收货地址
2. 联系收货人确认
3. 点击"确认送达"按钮
4. 订单状态更新为已完成

### 3.7 导航功能

**功能描述**: 提供配送路线导航

**功能特性**:

- 显示从商家到收货地址的路线
- 支持地图导航
- 显示预计配送时间

### 3.8 异常处理

**功能描述**: 处理配送过程中的异常情况

**异常类型**:

| 类型   | 说明       | 处理方式        |
| :--- | :------- | :---------- |
| 地址错误 | 收货地址无法找到 | 联系用户确认或联系商家 |
| 联系不上 | 无法联系收货人  | 等待或联系商家     |
| 商品问题 | 商品损坏或缺失  | 联系商家处理      |

## 4. 个人中心模块

### 4.1 个人信息

**功能描述**: 查看和修改个人资料

**信息字段**:

| 字段名  | 类型 | 说明     |
| :--- | :- | :----- |
| 骑手姓名 | 文本 | 骑手真实姓名 |
| 手机号  | 文本 | 联系电话   |
| 身份证号 | 文本 | 实名认证信息 |
| 头像   | 图片 | 个人头像   |
| 配送等级 | 文本 | 骑手等级标识 |

### 4.2 配送统计

**功能描述**: 展示骑手配送业绩

**统计内容**:

- 今日配送单数
- 本周配送单数
- 本月配送单数
- 累计配送单数
- 完成率
- 平均评分

### 4.3 收入明细

**功能描述**: 查看配送收入记录

**收入结构**:

- 配送费收入
- 奖励收入
- 扣除项（如违规罚款）
- 实发金额

### 4.4 消息通知

**功能描述**: 接收系统通知

**消息类型**:

- 订单消息：新订单提醒、订单取消通知
- 系统消息：公告、活动通知
- 个人消息：收入到账提醒

## 附录：订单状态流转

```
待接单 → 待取货 → 待配送 → 已完成
            ↓(取消)
           已取消
```

| 状态码 | 状态名称 | 说明           |
| :-: | :--- | :----------- |
|  1  | 待接单  | 订单已创建，等待骑手接单 |
|  2  | 待取货  | 骑手已接单，等待取货   |
|  3  | 待配送  | 已取货，正在配送中    |
|  4  | 已完成  | 订单已送达        |
|  0  | 已取消  | 订单已取消        |
