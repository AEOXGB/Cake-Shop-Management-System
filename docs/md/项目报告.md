## 1. 项目概述

蛋糕店管理系统是一个集商品展示、在线购物、订单管理、配送管理于一体的B2C电子商务平台。系统采用经典的MVC架构，前端使用ZUI组件库和Thymeleaf模板引擎，后端基于Spring Boot + MyBatis-Plus框架开发，支持用户端、管理端、骑手端三个角色使用，为蛋糕店提供完整的线上销售和配送解决方案。

### 1.1 项目背景

随着电子商务的快速发展，传统蛋糕店面临线上转型的需求。本系统旨在为蛋糕店提供一个功能完善、操作简便的线上销售平台，帮助商家扩大销售渠道，提升运营效率，同时为消费者提供便捷的在线购物体验。

### 1.2 系统角色

| 角色 | 说明 | 主要功能 |
|------|------|----------|
| 👤 普通用户 | 购买商品的消费者 | 浏览商品、加入购物车、下单支付、查看订单、个人信息管理 |
| 👑 管理员 | 店铺管理者 | 商品管理、分类管理、订单管理、用户管理、骑手管理、数据统计 |
| 🏍️ 骑手 | 配送人员 | 接单配送、订单状态更新、收入查看、个人信息管理 |
| 🤖 AI客服 | 智能客服 | 智能问答、商品咨询、售后支持 |

## 3. 系统架构

### 3.1 整体架构图

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   表现层     │ →   │   控制层     │ →   │   业务层     │ →   │   数据层     │
│ Thymeleaf+ZUI│     │ Spring MVC  │     │  Service    │     │ MyBatis-Plus │
│  HTML/CSS/JS │     │ Controller  │     │ ServiceImpl │     │   MySQL      │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
```

系统采用经典的MVC三层架构：
- **表现层**：负责页面展示和用户交互，使用Thymeleaf模板引擎
- **控制层**：接收请求，调用业务层，返回响应
- **业务层**：处理核心业务逻辑
- **数据层**：负责数据持久化，使用MyBatis-Plus操作MySQL

## 5. 数据库设计

系统共设计了 **12 张数据表**，涵盖用户、商品、订单、购物车、骑手等核心业务数据。数据库使用 MySQL 8.0，采用 UTF-8 编码，支持中文数据存储。

### 5.1 用户表 (user)

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| id | int | 用户ID | 主键，自增 |
| username | varchar(255) | 用户名 | 唯一 |
| password | varchar(255) | 密码 | 加密存储 |
| name | varchar(255) | 姓名 | - |
| email | varchar(255) | 邮箱 | - |
| phone | varchar(255) | 手机号 | - |
| address | varchar(255) | 地址 | - |
| isadmin | varchar(255) | 是否管理员 | yes/no |
| isvalidate | varchar(255) | 是否验证 | - |
| createtime | varchar(255) | 创建时间 | - |

### 5.2 商品表 (goods)

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| id | int | 商品ID | 主键 |
| name | varchar(255) | 商品名称 | - |
| cover | varchar(255) | 封面图 | - |
| image1 | varchar(255) | 详情图1 | - |
| image2 | varchar(255) | 详情图2 | - |
| price | varchar(255) | 价格 | - |
| intro | varchar(1000) | 商品介绍 | - |
| stock | int | 库存 | - |
| type_id | int | 分类ID | 外键 |
| daytime | varchar(255) | 上架时间 | - |

## 7. 部署说明

### 7.1 环境要求

| 软件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 1.8+ | Java开发环境 |
| MySQL | 8.0+ | 数据库服务 |
| Maven | 3.6+ | 项目构建工具 |
| Docker | 20.10+ | 容器化部署（可选） |

### 7.2 本地部署步骤

#### 步骤 1：克隆项目

```bash
git clone https://github.com/AEOXGB/Cake-Shop-Management-System.git
```

#### 步骤 2：创建数据库

创建名为 `cookieshop` 的数据库，执行 `docker/mysql/init.sql` 初始化数据

#### 步骤 3：修改配置

修改 `src/main/resources/application.yml` 中的数据库连接信息

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/cookieshop?characterEncoding=UTF-8&serverTimezone=UTC
    username: root
    password: your_password
```

#### 步骤 4：打包项目

```bash
mvn clean package -DskipTests
```

#### 步骤 5：运行项目

```bash
java -jar target/CakeShop-0.0.1-SNAPSHOT.jar
```

#### 步骤 6：访问系统

浏览器访问 `http://localhost:8090`

### 7.3 Docker 部署步骤

#### 步骤 1：安装 Docker 和 Docker Compose

确保服务器已安装 Docker 和 Docker Compose

#### 步骤 2：打包项目

```bash
mvn clean package -DskipTests
```

#### 步骤 3：启动服务

```bash
docker-compose up -d
```

#### 步骤 4：访问系统

浏览器访问 `http://服务器IP:8090`

### 7.4 阿里云部署步骤

#### 步骤 1：购买 ECS 服务器

在阿里云购买 ECS 实例，推荐配置：2核4G以上

#### 步骤 2：安装环境

安装 JDK、MySQL、Maven 或 Docker

#### 步骤 3：配置安全组

开放 8090 端口，允许外部访问

#### 步骤 4：部署项目

上传项目文件，启动应用服务

#### 步骤 5：域名解析（可选）

绑定域名，配置 Nginx 反向代理
