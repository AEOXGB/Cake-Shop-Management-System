## 一、本地 Docker 部署

### 1.1 环境准备

#### 步骤1：安装 Docker Desktop

1. 下载 Docker Desktop：[https://www.docker.com/products/docker-desktop/](https://www.docker.com/products/docker-desktop/)
2. 双击安装包进行安装
3. 安装完成后启动 Docker Desktop
4. 确保 Docker 图标显示在任务栏（Windows）或状态栏（Mac）

#### 步骤2：配置镜像加速器（非常重要！）

由于国内访问 Docker Hub 速度较慢，需要配置镜像加速器：

**Windows 系统配置方法：**

1. 右键点击任务栏中的 Docker 图标，选择「Settings」
2. 在左侧菜单中选择「Docker Engine」
3. 在配置框中添加以下内容：

```json
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me",
    "https://docker.m.daocloud.io",
    "https://docker.1panel.live",
    "https://hub.rat.dev",
    "https://docker.mirrors.ustc.edu.cn"
  ]
}
```

4. 点击「Apply & Restart」按钮重启 Docker

**验证配置是否生效：**

打开命令提示符，执行：
```bash
docker info
```

如果看到 `Registry Mirrors` 列表包含你配置的镜像地址，说明配置成功。

#### 步骤3：验证 Docker 安装

打开命令提示符（Windows）或终端（Mac），执行以下命令：

```bash
docker --version
docker-compose --version
```

如果显示版本信息，说明安装成功。

### 1.2 创建 Docker 配置文件

在项目根目录 `c:\1\CakeShop` 下创建以下文件：

#### 文件1：Dockerfile

```dockerfile
# 使用多阶段构建
FROM maven:3.8.6-openjdk-17 AS builder

# 设置工作目录
WORKDIR /app

# 复制 Maven 配置文件和源代码
COPY pom.xml .
COPY src ./src

# 编译项目，跳过测试
RUN mvn clean package -DskipTests

# 第二阶段：运行环境
FROM openjdk:17-jdk-slim

# 设置工作目录
WORKDIR /app

# 从构建阶段复制编译好的 JAR 文件
COPY --from=builder /app/target/*.jar app.jar

# 暴露端口
EXPOSE 8090

# 设置时区
ENV TZ=Asia/Shanghai

# 启动命令
ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 文件2：docker-compose.yml

```yaml
version: '3.8'

services:
  # MySQL 数据库服务
  mysql:
    image: mysql:8.0
    container_name: cakeshop-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: your_mysql_password  # 修改为你的密码
      MYSQL_DATABASE: cookieshop
      MYSQL_USER: cakeshop_user
      MYSQL_PASSWORD: cakeshop_password
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    networks:
      - cakeshop-network

  # CakeShop 应用服务
  cakeshop:
    build: .
    container_name: cakeshop-app
    restart: always
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/cookieshop?characterEncoding=UTF-8&serverTimezone=UTC
      SPRING_DATASOURCE_USERNAME: cakeshop_user
      SPRING_DATASOURCE_PASSWORD: cakeshop_password
    ports:
      - "8090:8090"
    depends_on:
      - mysql
    networks:
      - cakeshop-network

volumes:
  mysql_data:

networks:
  cakeshop-network:
    driver: bridge
```

#### 文件3：.dockerignore（可选）

```
target/
.mvn/
.git/
*.log
```

### 1.3 构建并启动容器

#### 步骤1：打开命令行

在 Windows 上打开 **命令提示符** 或 **PowerShell**，切换到项目目录：

```bash
cd c:\1\CakeShop
```

#### 步骤2：启动容器

执行以下命令启动所有服务：

```bash
docker-compose up -d
```

**说明**：
- `-d` 参数表示后台运行
- 第一次运行会下载镜像，可能需要几分钟
- Docker 会自动创建网络和数据卷

#### 步骤3：查看容器状态

```bash
docker-compose ps
```

如果显示 `cakeshop-mysql` 和 `cakeshop-app` 都是 `Up` 状态，说明启动成功。

### 1.4 验证本地部署

#### 步骤1：等待服务启动

容器启动后，应用需要一些时间初始化。等待约 30 秒后进行验证。

#### 步骤2：访问应用

打开浏览器，访问：

```
http://localhost:8090
```

如果能看到蛋糕系统首页，说明本地部署成功！

#### 步骤3：测试功能

1. 点击导航栏的各个链接
2. 尝试注册新用户
3. 浏览商品列表
4. 添加商品到购物车

## 二、阿里云上线准备

### 2.1 注册阿里云账号

1. 打开阿里云官网：[https://www.aliyun.com/](https://www.aliyun.com/)
2. 点击右上角「免费注册」
3. 按照提示完成账号注册和实名认证

### 2.2 申请免费试用 ECS

#### 步骤1：进入 ECS 控制台

1. 登录阿里云控制台：[https://ecs.console.aliyun.com/](https://ecs.console.aliyun.com/)
2. 如果没有开通 ECS，会提示开通，点击「立即开通」

#### 步骤2：申请免费试用

1. 在 ECS 控制台首页，找到「免费试用」入口
2. 选择「云服务器 ECS」免费试用
3. 配置信息：
   - **地域**：选择离你最近的区域（如上海、北京）
   - **实例规格**：选择免费规格（如 `ecs.t6-c1m1.small`）
   - **操作系统**：选择 Ubuntu 22.04 LTS 或 CentOS 7.x
   - **购买时长**：选择 1 个月

#### 步骤3：设置登录密码

**非常重要！** 在创建实例时设置登录密码：
1. 在「实例配置」页面找到「登录凭证」
2. 选择「自定义密码」
3. 设置一个安全的密码（建议包含大小写字母、数字、特殊字符）
4. 记住这个密码，后续登录服务器需要使用

### 2.3 配置安全组

#### 步骤1：找到安全组配置

1. 在 ECS 控制台左侧菜单，点击「安全组」
2. 选择你创建的实例所在的安全组

#### 步骤2：添加安全规则

点击「添加安全规则」，添加以下规则：

| 规则方向 | 端口范围 | 授权对象 | 说明 |
|:---|:---|:---|:---|
| 入方向 | 22 | 0.0.0.0/0 | SSH 登录 |
| 入方向 | 80 | 0.0.0.0/0 | HTTP 访问 |
| 入方向 | 443 | 0.0.0.0/0 | HTTPS 访问 |
| 入方向 | 8090 | 0.0.0.0/0 | 应用端口 |
| 入方向 | 3306 | 0.0.0.0/0 | MySQL 端口（生产环境建议限制 IP） |

## 三、阿里云服务器配置

### 3.1 登录 ECS 服务器

#### 方法1：使用阿里云控制台登录

1. 在 ECS 控制台，找到你的实例
2. 点击「远程连接」
3. 选择「Workbench 远程连接」
4. 输入登录密码，点击「确定」

#### 方法2：使用 SSH 工具登录（推荐）

**Windows 用户**：使用 PuTTY 或 Windows Terminal
**Mac/Linux 用户**：使用终端

```bash
ssh root@你的服务器公网IP
```

输入之前设置的密码即可登录。

### 3.2 安装 Docker

#### 步骤1：更新系统

```bash
# Ubuntu 系统
apt update && apt upgrade -y

# CentOS 系统
yum update -y
```

#### 步骤2：安装 Docker（Ubuntu）

```bash
# 安装依赖
apt install -y apt-transport-https ca-certificates curl software-properties-common

# 添加 Docker GPG 密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 添加 Docker 源
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker
apt update && apt install -y docker-ce docker-ce-cli containerd.io

# 启动 Docker 服务
systemctl start docker
systemctl enable docker

# 验证安装
docker --version
```

#### 步骤2：安装 Docker（CentOS）

```bash
# 安装依赖
yum install -y yum-utils device-mapper-persistent-data lvm2

# 添加 Docker 源
yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 安装 Docker
yum install -y docker-ce docker-ce-cli containerd.io

# 启动 Docker 服务
systemctl start docker
systemctl enable docker

# 验证安装
docker --version
```

#### 步骤3：安装 Docker Compose

```bash
# 下载 Docker Compose
curl -SL https://github.com/docker/compose/releases/download/v2.20.0/docker-compose-linux-x86_64 -o /usr/local/bin/docker-compose

# 添加执行权限
chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

### 3.3 安装 MySQL（可选，也可以用 Docker）

**方案A：使用 Docker MySQL（推荐）**

在后续步骤中通过 docker-compose 自动安装。

**方案B：安装本地 MySQL**

```bash
# Ubuntu
apt install -y mysql-server

# CentOS
yum install -y mysql-server

# 启动服务
systemctl start mysql
systemctl enable mysql
```

## 四、项目部署到阿里云

### 4.1 上传项目代码

#### 方法1：使用 Git 克隆

```bash
# 创建项目目录
mkdir -p /opt/cakeshop
cd /opt/cakeshop

# 克隆代码（如果有 Git 仓库）
git clone <你的仓库地址> .

# 如果没有 Git 仓库，可以使用方法2
```

#### 方法2：使用 FTP 上传

1. 在本地打包项目：
   ```bash
   cd c:\1\CakeShop
   zip -r cakeshop.zip .
   ```

2. 使用 FileZilla 或 WinSCP 上传 `cakeshop.zip` 到服务器

3. 在服务器解压：
   ```bash
   mkdir -p /opt/cakeshop
   unzip cakeshop.zip -d /opt/cakeshop
   cd /opt/cakeshop
   ```

### 4.2 配置环境变量

编辑 `docker-compose.yml`，修改数据库密码：

```bash
nano docker-compose.yml
```

修改以下内容：
- `MYSQL_ROOT_PASSWORD`: 设置一个安全的密码
- `MYSQL_PASSWORD`: 设置数据库用户密码
- `SPRING_DATASOURCE_PASSWORD`: 与上面一致

按 `Ctrl+O` 保存，按 `Ctrl+X` 退出。

### 4.3 构建并启动容器

```bash
# 进入项目目录
cd /opt/cakeshop

# 构建并启动容器
docker-compose up -d
```

**注意**：第一次运行需要下载镜像和构建，可能需要 5-10 分钟。

### 4.4 验证上线

#### 步骤1：查看容器状态

```bash
docker-compose ps
```

#### 步骤2：查看应用日志

```bash
docker-compose logs cakeshop
```

如果看到 `Started CakeShopApplication`，说明启动成功。

#### 步骤3：访问应用

打开浏览器，访问：

```
http://你的服务器公网IP:8090
```

如果能看到蛋糕系统首页，说明上线成功！

#### 步骤4：测试功能

1. 点击导航栏的各个链接
2. 尝试注册新用户
3. 浏览商品列表
4. 添加商品到购物车
5. 测试管理员后台

## 五、常见问题排查

### 问题1：Docker 容器启动失败

**现象**：`docker-compose ps` 显示容器状态为 `Exit`

**解决方法**：
```bash
# 查看容器日志
docker-compose logs cakeshop

# 常见原因：
# 1. 数据库连接失败 - 检查数据库配置
# 2. 端口被占用 - 检查 8090 端口
# 3. 代码编译错误 - 检查 Maven 编译日志
```

### 问题2：无法访问应用

**现象**：浏览器显示无法访问

**解决方法**：
1. 检查安全组是否开放了 8090 端口
2. 检查服务器防火墙是否开放了端口：
   ```bash
   # Ubuntu
   ufw status
   ufw allow 8090
   
   # CentOS
   firewall-cmd --list-all
   firewall-cmd --add-port=8090/tcp --permanent
   firewall-cmd --reload
   ```

### 问题3：数据库连接失败

**现象**：应用日志显示 `Cannot get connection`

**解决方法**：
1. 检查 `docker-compose.yml` 中的数据库配置
2. 确保数据库容器先启动：
   ```bash
   docker-compose up -d mysql
   # 等待 10 秒后启动应用
   docker-compose up -d cakeshop
   ```

### 问题4：Docker 命令权限不足

**现象**：`docker: Got permission denied`

**解决方法**：
```bash
# 将当前用户添加到 docker 组
sudo usermod -aG docker $USER

# 重新登录或执行
newgrp docker
```

## 六、附录

### A. 常用命令速查表

```bash
# Docker 相关
docker ps                    # 查看运行中的容器
docker ps -a                 # 查看所有容器
docker stop <容器名>          # 停止容器
docker start <容器名>         # 启动容器
docker rm <容器名>            # 删除容器
docker logs <容器名>          # 查看容器日志
docker-compose up -d         # 启动所有服务
docker-compose down          # 停止并删除容器
docker-compose restart       # 重启服务

# 服务器相关
systemctl status docker      # 查看 Docker 状态
systemctl restart docker     # 重启 Docker 服务
netstat -tlnp                # 查看端口占用
```

### B. 文件结构说明

```
CakeShop/
```
